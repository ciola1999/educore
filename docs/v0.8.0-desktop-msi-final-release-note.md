## Ringkasan

EduCore desktop production sekarang memakai strategi `embedded local web server` yang dijalankan oleh startup manager native Tauri. Window utama hanya dibuka setelah runtime loopback sehat, dan auth packaged desktop sudah lolos smoke nyata pada installer MSI.

Release ini menutup blocker packaged desktop yang sebelumnya paling besar:
- runtime desktop production yang tadinya scaffold
- jendela `bun` yang ikut muncul
- login MSI yang gagal atau menggantung
- relogin setelah logout dalam sesi app yang sama
- mismatch antara auth desktop lokal dan middleware server
- stale runtime cache yang membuat hasil build terlihat tidak konsisten

## Yang Sudah Lulus

- install MSI di Windows target
- login desktop dengan akun inti
- logout lalu login ulang tanpa restart app
- dashboard terbuka
- students terbuka
- teachers terbuka
- courses terbuka
- attendance terbuka
- settings terbuka
- full sync online berhasil
- full sync offline gagal dengan pesan jujur saat internet tidak tersedia

## Perubahan Teknis Utama

- embedded runtime production sudah dipaketkan sebagai bundle nyata
- startup manager native menangani `spawn / probe / stop`
- Tauri window menunggu health check loopback sebelum navigasi ke app
- auth desktop dibuat native-first dan fail-fast
- middleware loopback memakai proof runtime server-side yang stabil
- logout tidak lagi merusak proof runtime desktop
- desktop build pipeline menghasilkan artifact MSI yang konsisten
- runtime packaged sekarang memakai archive stabil `runtime-bundle.tar`
- tersedia jalur recovery resmi `bun run build:desktop:msi-finalize` untuk host Windows yang timeout pada `bun tauri build`

## Residual Risk yang Masih Terbuka

- installer desktop tetap controlled artifact dan harus diperlakukan sensitif
- smoke install nyata pada mesin bersih tetap perlu diulang setelah hardening credential first-run ini
- modularisasi `desktop-local-api.ts` masih bisa diteruskan agar audit auth/sync lebih mudah
- hash final artifact perlu dicatat ulang di mesin release yang punya utilitas hashing standar
- `bun tauri build` penuh di host agent Windows masih rawan timeout, jadi jalur finalisasi MSI pasca-WiX perlu tetap didokumentasikan

## Recovery Build Path

Jika `bun tauri build` sudah berhasil sampai menghasilkan snapshot `wix/x64` tetapi gagal/timeout sebelum MSI final selesai, gunakan jalur berikut:

1. jalankan `bun run build:desktop`
2. jalankan `bun tauri build` sejauh host mengizinkan
3. jika snapshot `src-tauri/target/release/wix/x64/main.wxs` sudah ada, jalankan `bun run build:desktop:msi-finalize`

Catatan:
- jalur ini menyelaraskan `runtime-config.json` ke `runtime-bundle.tar`
- jalur ini membersihkan referensi `runtime-bundle-<timestamp>.tar` dari snapshot WiX
- jalur ini compile ulang `main.wixobj` dan link MSI dengan `light.exe -sval`
